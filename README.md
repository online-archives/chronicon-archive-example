# Chronicon Archive Example

Live demo of [Chronicon](https://github.com/19-84/chronicon) archiving the [Discourse Meta Theme category](https://meta.discourse.org/c/customization/theme/61/).

**Live site:** https://online-archives.github.io/chronicon-archive-example

## What this is

A static archive of the Discourse Meta forum's Theme customization category, generated by Chronicon and hosted on GitHub Pages. The archive includes:

- Static HTML with full navigation and client-side search
- Markdown version for browsing directly on GitHub
- SEO metadata (Open Graph, Twitter Cards, JSON-LD, sitemap.xml)

## How it works

A [GitHub Actions workflow](.github/workflows/update-archive.yml) runs weekly to:

1. Restore the SQLite database from cache
2. Run an incremental update (fetches only new/modified posts)
3. Re-export the archive in hybrid format (HTML + Markdown)
4. Deploy the static files to the `gh-pages` branch

## Local usage

### Quick start (no images, ~2 minutes)

```bash
make archive-sqlite-fast
```

### Full archive with images (~2 hours)

```bash
make archive-sqlite
```

The Theme category has ~3000 screenshot images. Use `*-fast` targets for validation, full targets for production archives.

### Archive with PostgreSQL

```bash
cp .env.example .env
make archive-postgres-fast   # no images
make archive-postgres        # with images
```

### Incremental update

```bash
make update
```

### Re-export from existing database

```bash
make export
```

### Full E2E validation (both backends)

```bash
make e2e
```

Runs fast mode (no images) against both SQLite and PostgreSQL backends.

### View locally

After archiving, open `archives/index.html` in a browser.

### Cleaning up

```bash
make clean
```

Uses the Docker container for cleanup to handle file ownership correctly.

## Configuration

See [`.chronicon.toml`](.chronicon.toml) for the archive configuration. Key settings:

- **Target:** meta.discourse.org, Category 61 (Theme)
- **Format:** Hybrid (HTML at root + Markdown in `/md/`)
- **Search:** Static client-side search
- **Canonical URL:** https://online-archives.github.io/chronicon-archive-example

## Requirements

- Docker and Docker Compose
- Disk space: ~5MB without images, ~200MB with images
